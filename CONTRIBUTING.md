# Contributing to Termbackup

Thank you for your interest in contributing to Termbackup! We welcome all contributions, from bug reports to pull requests.

## Getting Started

1. Fork the repository
2. Clone locally and navigate to the project root
3. Install dependencies: `pip install -e .`
4. Make your changes and test them thoroughly
5. Ensure code passes linters
6. Submit a Pull Request targeting the `main` branch!

## Architecture

Termbackup v2 is a zero-trust cryptographic matrix engine. Please respect the existing architecture patterns, especially concerning zero trust, encryption, and the lack of external dependency bloating.
